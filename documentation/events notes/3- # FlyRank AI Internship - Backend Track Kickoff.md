# FlyRank AI Internship - Backend Track Kickoff

Welcome to the **FlyRank AI Backend Track**! This guide summarizes the kickoff presentation and track expectations led by your mentor, Harris Hojich [00:00:03, 00:01:38]. 

The primary goal of this track is to bridge the gap between abstract university theory and real-world system patterns [00:04:47]. We emphasize core architectural **concepts over technology choices** [00:07:50, 00:12:22].

---

## 💡 Core Philosophy & AI Integration
* **Concepts > Language Syntax:** At this level, individual language syntax is minimal because AI can generate structure seamlessly [00:09:13]. Your priority is to understand optimal data flows and design solutions [00:09:34].
* **AI as a Companion:** Use Large Language Models (LLMs) to share architectural ideas, probe concepts, and study documentation, rather than copying code blindly [00:24:32]. You must **own every single line** you build [00:12:34, 00:25:14].
* **The "Let's Figure It Out Together" Mindset:** Dealing with imposter syndrome is completely normal [00:02:18]. Engineering operates in continuous learning environments where sharing ideas with your team drives success [00:03:03, 00:03:12].
* **The Importance of Attention:** In an automated world, cultivating focused attention—reading documentation properly and understanding structural dependencies—is a critical engineer differentiator [00:04:12, 00:34:39].

---

## 🛠️ Track Parameters & Logistics
* **Real-World Problem Scope:** Due to data privacy, customer secrets, and high-volume security concerns, interns will not interface directly with FlyRank's live codebase [00:05:47, 00:06:02]. Instead, you will engineer a **Capstone Project** mimicking actual production feature designs [00:06:18, 00:06:27].
* **Technology Agnostic Stack:** While FlyRank uses a core stack of **Next.js and TypeScript**, you are completely free to build your projects using **Python** or any language you are proficient in [00:08:44, 00:29:26]. Choose one stack early and stick with it [00:09:03].
* **Public GitHub Portfolios:** All assignments must be developed inside your own public GitHub repositories and linked back to the portal for completion tracking [00:28:29].
* **Free Tier Infrastructure:** Every mandatory requirement can be fulfilled using completely free services—zero personal spend is required [00:06:56].

## 📅 The Backend Weekly Rhythm
1. **Live Synchronous Events:** 1–2 technical lectures, guest developer workshops, or workflow walkthroughs per week [00:09:52, 00:37:44].
2. **Weekly Assignments:** Iterative practical mini-projects serving as direct building blocks toward your final capstone feature [00:10:16].
3. **Curated Reading & Resources:** Post-lecture articles, official documentation, and reference video packets are updated weekly in your portal profile [00:11:11, 00:28:56].

## 🚀 Technical Road Map

| Week | Phase / Session | Core Topics Covered |
| :--- | :--- | :--- |
| **Week 1** | **Introduction & Setup** | Overview of the system roadmap, configuring your environment, and launching Assignment 1 (Basic HTTP API creation with customized error/status handling) [00:13:30, 00:30:55]. |
| **Week 2** | **JavaScript & Web 101** | Deep dive into JS history, Node runtime internals, request/response models, servers vs. clients, API mechanisms, HTTP mechanics, and system frames [00:13:47, 00:14:20]. |
| **Week 3** | **Capstone Reveal & Architecture** | Reviewing defined capstone paths and structural requirements. Launching scalable, layered Next.js design patterns (separating services from repositories) [00:15:03, 00:15:44]. |
| **Week 4** | **Survival Kit & Security** | Context handling using Git, Docker, local SQL databases, and environment configs [00:16:51, 00:17:21]. Transitioning to Authentication vs. Authorization, JVT tokens, and `401`/`403` handling [00:17:46]. |
| **Week 5** | **AI Engineering & Workflows** | Context and prompt engineering with ML Mentor Mirza [00:18:17]. Managing durable background jobs, queue managers, workers, and long-running LLM tasks utilizing tools like `Inngest` [00:18:48, 00:19:23]. |

| **Week 8** | **Deployment & Observability** | Shipping your product, managing security layers, evaluating live logging architectures, and wrapping up your capstone project [00:22:15, 00:22:24]. |
